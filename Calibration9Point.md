## 先理解"九点标定"在做什么

想象一个场景：你有一个摄像头俯视一块工作台，你想知道**"图像里的某个像素点，对应工作台上的哪个真实位置"**。

```
摄像头图像（像素坐标）        工作台（真实坐标/mm）
    
  (100,100)●──●──●            ●──────●──────●
           │  │  │            0mm   150mm  300mm
  (100,400)●──●──●     ─→     │      │      │
           │  │  │            ●──────●──────●
  (100,700)●──●──●           150mm  ...    ...
                              │      │      │  
                              ●──────●──────●
                             300mm  ...    ...
```

九点标定就是：**用9个已知的对应点，让程序学会这种坐标转换规则**，然后就能转换任意点了。

## 代码分3步走

### 第一步：提供9对对应点

```cpp
// 我在图像里测量/点击了9个位置（像素）
std::vector<cv::Point2f> imagePoints = {
    {100, 100}, {400, 100}, {700, 100},   // 第一行3个点
    {100, 400}, {400, 400}, {700, 400},   // 第二行3个点
    {100, 700}, {400, 700}, {700, 700}    // 第三行3个点
};

// 我知道这9个位置在工作台上的真实坐标（毫米）
std::vector<cv::Point2f> worldPoints = {
    {0, 0}, {150, 0}, {300, 0},
    {0, 150}, {150, 150}, {300, 150},
    {0, 300}, {150, 300}, {300, 300}
};
```

这就是"老师给的答案"——9道题，每题都知道输入和输出。

---

### 第二步：让 OpenCV 找出规律（求矩阵 H）

```cpp
H = cv::findHomography(imagePoints, worldPoints, 0);
```

这一行做的事情是：**根据9对点，计算一个3×3的变换矩阵H**。

H 矩阵长这样：
```
H = | h00  h01  h02 |
    | h10  h11  h12 |
    | h20  h21  h22 |
```

你不需要手动算，OpenCV 帮你解方程组。

---

### 第三步：用 H 矩阵转换任意新点

```cpp
cv::Point2f world = calib.pixelToWorld(testPixel);
```

内部调用了：
```cpp
cv::perspectiveTransform(src, dst, H);
// 本质是矩阵乘法: dst = H × src
```

## 用一张图总结整个流程

```
         【标定阶段】（只做一次）
         
  9个图像点  ──┐
               ├──→ findHomography ──→ 矩阵 H（保存起来）
  9个真实点  ──┘

         【使用阶段】（反复使用）
         
  任意图像点 ──→ H × 点 ──→ 真实坐标  ✓
  任意真实点 ──→ H⁻¹ × 点 ──→ 图像坐标 ✓
```

---

## OpenCV 你只需要知道这几个函数

| 函数 | 通俗理解 |
|------|---------|
| `cv::Point2f(x, y)` | 就是一个坐标点 (x, y) |
| `cv::findHomography(src, dst)` | 给一堆点对，求变换矩阵 |
| `cv::perspectiveTransform(src, dst, H)` | 用矩阵H批量转换坐标点 |
| `cv::FileStorage` | 把矩阵存到文件/从文件读出来 |

